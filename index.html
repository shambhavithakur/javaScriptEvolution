The Evolution of

JavaScript

Learn how JavaScript evolved from a simple programming language for websites to a powerful workhorse that is part of web pages and various other types of client- and server-side applications.

JS

1995–1998

1995

JavaScript Created

JavaScript was created and put to use in 1995 at Netscape Communications Corporation. Netscape was a computer services company that owned Netscape Navigator, a popular '90s web browser.

The source code of JavaScript was the brainchild of a Netscape employee, Brendan Eich. Brendan wrote the code in 1995 with the purpose of making web pages interactive. The Netscape Navigator browser was updated concurrently to include a dedicated JavaScript engine. This engine allowed JavaScript-based content to run in web pages.

1996

Standardizing Body Formed

Because of the growing popularity of JavaScript, an international body called European Computer Manufacturers Association (ECMA) was formed in 1996 to standardize the language and manage its development.

1997

ECMAScript Released

In 1997, ECMA released ECMAScript to serve as a standardized specification for JavaScript and any similar languages. Since then, ECMAScript has been continually revised and updated to meet the changing needs of developers and businesses. At present, ECMA revises ECMAScript every year.

1998

Minor Changes with ECMAScript 2

The second version of ECMAScript, called ECMAScript 2, was released in 1998. This version contained minor editorial changes, such as error corrections and revised wording. No new features and functions were specified in this version.

1999–2002

1999

Groundwork Built with ECMAScript 3

ECMAScript 3, the third version of ECMAScript, was released in 1999. This version provided the groundwork for the JavaScript that we use today. Significant changes in this version were the inclusion of try/catch statements and regular expressions.

2000

ECMAScript 4 Project Initiated

A project focusing on the development of the fourth version of ECMAScript, ECMAScript 4, was initiated in 2000. This project received support from Microsoft, which wanted to incorporate ECMAScript 4 specifications in JScript, a language that it had developed in house. JScript had similar functionality as JavaScript and had been specially created for Internet Explorer.

2003–2006

2003

ECMAScript 4 Project Scrapped

The ECMAScript 4 project was scrapped in 2003 because of differences between Microsoft and ECMA. Microsoft was reluctant to include JavaScript in Internet Explorer and, probably, wanted an overhaul that would have changed the fundamental principles of the language.

2005

Ajax Introduced

Ajax or asynchronous JavaScript was introduced to the developer community in 2005. Ajax made it possible to create dynamic applications that could process and load data in a web page without requiring a full reload of the page.

2007–2010

2007

ECMAScript 3.1 on the Horizon

Microsoft and its partners began work on ECMAScript 3.1 in 2007. In this version of ECMAScript, they wanted to incorporate the standards that had been rejected during the ECMAScript 4 project.

2008

Future of ECMAScript Defined

In 2008, Microsoft and a few other factions that had a stake in ECMAScript met in Oslo to settle their disputes. This meeting was considered a success because the participants reached a consensus on the future course of ECMAScript.

2009

ECMAScript 5 and Node.js Released

ECMAScript 3.1 was renamed as ECMAScript 5 and released to the world in 2009. This fifth version of ECMAScript introduced support for JavaScript Object Notation (JSON) and the strict mode. JSON is a format for structuring data and transferring it between client- and server-side applications. The strict mode allows for improved error checking in code.

In an event unrelated to ECMA, Ryan Dahl wrote Node.js, which is a JavaScript runtime environment. Node.js can run JavaScript code on the server side.

2010

NPM Written

Node package manager(npm), which, as its name suggests, is a package manager for Node.js, was written in 2010. Npm simplifies the process of installing, uninstalling, and updating Node.js libraries.

2011–2014

2011

ECMAScript 5.1 and a Windows Version of Node.js

The differentiating factor between ECMAScript 5.1, which was released in 2011, and its predecessor was editorial changes. These changes were introduced to meet all the requirements stipulated by the third edition of the ISO/IEC 16262 international standard. ECMAScript 5.1 didn't specify any new features and functions.

Another significant development in 2011 was in the Node.js area. A Windows version of Node.js was released this year, making it possible for Windows applications to use the powerful features of Node.js.

2012

React Written

Jordan Walke, a Facebook employee, wrote React in 2012. React is an open-source JavaScript framework for building user interfaces (UIs).

React was initially meant to be used in the Facebook Ads application. However, owing to its smilicity, scalability, and fast throughput, React has now been adopted by users outside Facebook.

If you want to create a large data-based web application whose UIs need to be updated in real time, you can consider using React. You can render React-based UIs on the client side in web browsers or on the server side with Node.js.

2015–2018

2015

ECMAScript 2015 and a Protocol Change

ECMAScript 2015, the sixth version of ECMAScript, was released in June 2015. The most significant change in this release was the inclusion of classes and modules.

The year 2015 also saw a change in the naming convention of ECMAScript versions. From 2015 onwards, ECMA began to suffix the year of release rather than the version number to the names of ECMAScript versions—therefore, ECMAScript 2015 instead of ECMAScript 6. This change was a corollary of ECMA's decision to release an updated version of ECMAScript each year.

2016

Power Added with ECMAScript 2016

The seventh version of ECMAScript, ECMAScript 2016, was released in June 2016. Various powerful features and functions were included in this version.

2017

Power Enhanced with ECMAScript 2017

ECMAScript 2017, the eighth version of ECMAScript, was released in June 2017. Many powerful features and functions were added to this version.
